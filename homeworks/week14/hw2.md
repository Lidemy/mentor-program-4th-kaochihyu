#### 部署

最一開始註冊好 AWS 後，我連要選哪一個都不知道，太多選項我搞不清楚到底該幹嘛，後來瞄到推薦文章的標題，才大概知道是跟 EC2 有關

我找到 EC2，也看著教學文件一步步照著指示去設定，但當執行到了遠端桌面連線的時候就出現問題了，我試著連線卻出現：
遠端桌面無法連線到遠端電腦，造成此問題的原因可能是下列之一:

1. 未啟用對伺服器的遠端存取
2. 遠端電腦已關機
3. 遠端電腦未連線到網路

所以試著去檢查系統防火牆的設定，看遠端桌面的公用、私人是否被勾選，也試著將網路重新設定，都沒有用，最終還是去看了別人的作法，才發現其他人貌似都用 Ubuntu，我用的是 Windows，連線的操作跟 Linux 不太一樣，後來改成 Ubantu 的設定後，連線就沒有之前那麼複雜的問題了

LAMP 之後的步驟參考了同學筆記跟講解影片，第一次跟著同學的筆記做的時候有很多地方都不知道為甚麼要那樣做，就只是照著做而已，後來跟著影片再做的時侯，比較知道那些指令為甚麼要那樣打，還有那些設定代表著甚麼，對於整體理解也算是滿有幫助的，是說中間有點問題，就是第二次要在本地重新連上主機時，不知道為甚麼突然要我登入密碼，明明昨天也不用，結果我輸入了我以為的密碼，然後顯示密碼錯誤，試了好幾個，輸入三次錯誤就要再重新輸入金鑰，累積三次金鑰都輸入錯誤，連線就被擋了

我想說那就重新 launch 一個 instance 好了，結果我發現我最初的那個主機的 IP 變了，於是用了這個變了的 IP 再連一次主機，這次不用密碼就連上了，不確定是不是跟我剛剛改變防火牆設定有關

明明叫我們自己部署，卻還是看了別人寫好的步驟來完成，老實說感覺不太好，好像是靠作弊來完成測驗一樣，但是說真的光是看著別人寫好的步驟做我都覺得很難了，何況我在 AWS 裡要去選擇 EC2 這麼一個超超超初步的步驟我都折騰了一番
